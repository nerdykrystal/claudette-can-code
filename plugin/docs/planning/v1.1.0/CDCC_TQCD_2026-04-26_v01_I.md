## 2. Testing Taxonomy Applicability

### 2.1 Test Categories (Part 1)

### TQCD-TC-01: Unit Tests
- Applies: yes. Exit: 100% line + branch on `src/core/**` and `src/hooks/**`. Vitest. Excluded: schema constants.

### TQCD-TC-02: Integration Tests
- Applies: yes. Exit: every cross-module boundary (CLI ↔ core ↔ hooks ↔ sqlite ↔ filesystem) has at least one integration test. Real sqlite + real fs (no mocks).

### TQCD-TC-03: Component / Module Tests
- Applies: yes. Exit: each AVD-AC has its own component test file under `tests/component/`.

### TQCD-TC-04: End-to-End Tests
- Applies: yes. Exit: `cdcc generate` → hook fires → audit entry queryable, executed against a real bundle fixture (the CDCC v1.1.0 bundle itself).

### TQCD-TC-05: Functional Tests
- Applies: yes. Exit: each TRD-FR-NN has at least one functional test row in §5.2.

### TQCD-TC-06: Regression Tests
- Applies: yes. Exit: every gate-22 finding has a regression test that fails on the v1.0.4 baseline and passes on v1.1.0 (test_path in §5.2).

### TQCD-TC-07: Smoke Tests
- Applies: yes. Exit: `cdcc generate --dry-run` on a sample bundle in CI smoke job; <30s wall.

### TQCD-TC-08: Sanity Tests
- Applies: yes. Exit: every released artifact passes `cdcc audit --since=<install-ts>` returning ≥1 row.

### TQCD-TC-09: Acceptance Tests (UAT)
- Applies: yes. Exit: practitioner runs CDCC v1.1.0 against a 5-doc bundle and observes PRD-UJ-01/02/03 outcomes.

### TQCD-TC-10: Contract Tests (API-level)
- Applies: yes. Exit: schemas/*.schema.json validate every emitted artifact (plan, plan-state, audit-entry, recovery-event, attempted-violation).

### TQCD-TC-11: Performance Tests
- Applies: yes. Exit: TRD-NFR-3.1-01 / -02 / -03 budgets met in CI perf job. k6 not applicable (no HTTP); custom Node bench harness with statistical-aggregation (median + p95 + p99 across 100 runs).

### TQCD-TC-12: Security Tests
- Applies: yes. Exit: gitleaks pre-commit; npm audit (high+critical fails CI); semgrep static analysis (severity ≥ ERROR fails CI).

### TQCD-TC-13: Usability Tests
- Applies: limited. Exit: developer dogfooding (Krystal authoring this very bundle's downstream build is the usability test).

### TQCD-TC-14: Accessibility Tests
- Skip reason: no UI surface (per PRD §6.4 / TRD §3.5).

### TQCD-TC-15: Snapshot / Visual Regression
- Skip reason: no UI.

### TQCD-TC-16: Property-Based / Fuzz Tests
- Applies: yes. Exit: bundle-hash collision-resistance property (FR-14); audit-log entry round-trip property (write→read returns identical entry).

### TQCD-TC-17: Mutation Testing
- Applies: yes. Exit: stryker mutation score ≥ 80% on `src/core/**`. Threshold matches existing stryker.conf.mjs.

### TQCD-TC-18: CI/CD Pipeline Tests
- Applies: yes. Exit: every PR runs lint + typecheck + unit + integration + e2e + perf + security + mutation; merge protection enabled.

### TQCD-TC-19: Canary / A/B Testing
- Skip reason: single deploy target; npm package install is binary on/off.

### TQCD-TC-20: Monitoring / Observability (post-deploy)
- Applies: yes. Exit: audit DB itself IS the observability target (per TRD-NFR-3.8-01); no remote monitoring.
