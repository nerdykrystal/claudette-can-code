# PRD — Hello World SvelteKit App

## Purpose

Minimal application used as the target of the Sandbox Validation Harness. The app itself is not a product — it is an instrument for verifying that the Sandbox Setup Protocol + D2R pipeline + measurement tooling all function end-to-end.

## Product Summary

A single static web page that displays a greeting message.

## User Story

As a visitor to the Hello World site, I see a visible greeting message ("Hello, World!") rendered on the page when I load the URL.

## Functional Requirements

- **FR-1** — The app is a SvelteKit 5 static site (no server runtime needed).
- **FR-2** — The app has exactly one page at route `/`.
- **FR-3** — The page renders exactly one component, `<Greeting />`.
- **FR-4** — `<Greeting />` displays the text "Hello, World!" visibly on the page.
- **FR-5** — `<Greeting />` uses at least one Svelte 5 reactive hook (a `$effect` rune or equivalent).
- **FR-6** — Build output is pure static HTML/CSS/JS suitable for any static host.

## Non-Functional Requirements

- **NFR-1** — WCAG 2.1 AA accessibility compliance. The greeting must be reachable to assistive technologies.
- **NFR-2** — Lighthouse scores: Performance ≥95, Accessibility 100, Best Practices ≥95, SEO ≥90.
- **NFR-3** — Zero console errors on page load.
- **NFR-4** — Total bundle size < 100 KB (trivial).
- **NFR-5** — Type-checks clean under TypeScript strict mode.

## Out of Scope

- Internationalization
- Dynamic content or API calls
- User interaction beyond page load
- Authentication
- Backend services
