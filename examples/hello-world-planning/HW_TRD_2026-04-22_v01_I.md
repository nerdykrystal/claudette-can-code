# TRD — Hello World SvelteKit App

## Stack (Specific Versions)

- **Framework:** SvelteKit 5.x (latest stable as of 2026-04-22)
- **UI Framework:** Svelte 5.x with runes (`$state`, `$effect`, `$derived`)
- **Language:** TypeScript 5.7.x, strict mode
- **Test Framework:** Vitest 1.x with `@vitest/ui` optional
- **Static Adapter:** `@sveltejs/adapter-static`
- **Build Tool:** Vite 5.x (bundled with SvelteKit)
- **Accessibility Testing:** `@axe-core/playwright` or `axe-playwright` for integration tests; `axe-core/cli` for post-build audits
- **Linting:** ESLint 9.x with `eslint-plugin-svelte` and TypeScript parser
- **Formatting:** Prettier with `prettier-plugin-svelte`
- **Coverage:** `c8` via Vitest built-in

## API Surface

None. Static site. No server runtime.

## Data Model

None. Greeting text is a hardcoded string literal.

## Build Process

1. `npm install` — installs dependencies
2. `npm run build` — produces `build/` directory with static HTML/CSS/JS
3. `npm run preview` — serves `build/` for measurement-tool access

## Deployment Target

Static host. For validation-harness purposes, served locally via `npm run preview` on port 4173 (SvelteKit default) for Lighthouse + axe-core scans.

## Performance Budget

- First Contentful Paint < 500 ms on local preview
- Time to Interactive < 500 ms
- Total JS bundle < 50 KB (Svelte 5's runtime is lean)
- Total CSS < 10 KB

## Security Considerations

- No user input → no XSS/SQL injection surface
- No server → no CSRF/SSRF surface
- Static host → no secret management needed in the app itself

## IP-Clean Naming Requirements

Per `_grand_repo/docs/Pre_Publication_IP_Scrub_Checklist_2026-04-22_v02_I.md`:
- Folder names: branded terminology only
- Commit messages: branded terminology only
- Test names: `test_greeting_renders_hello_world` is fine; avoid any methodology-revealing test names
- Identifier names: no internal-process names like `selfAuditEdit*`

## Acceptance Test Integration

Vitest runs as part of CI. Every build that reaches Stage 03 must have passing Vitest results. The one test in `Greeting.test.ts` is the minimum; D2R pipeline may add more during QA.
