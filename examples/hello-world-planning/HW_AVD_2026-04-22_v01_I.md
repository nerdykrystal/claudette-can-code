# AVD — Hello World SvelteKit App

## Status Declaration

This is a **Skipped-Status AVD** per D2R spec for trivially simple projects. The app has no meaningful architecture to vision — one component, one page, static output. The AVD exists to satisfy the D2R pipeline's prerequisite-document requirement and to confirm that the pipeline accepts Skipped-Status AVDs.

## Architecture Summary

Monolithic single-page static site. No architectural decomposition needed.

## Data Flow

```
Browser loads +page.svelte
  → +page.svelte renders <Greeting />
  → <Greeting /> mounts, $effect runs (logs to console or sets internal state)
  → User sees "Hello, World!"
```

That is the entire data flow.

## Component Boundaries

| Component | Responsibility |
|---|---|
| `+layout.svelte` | HTML scaffold (head tags, root div) |
| `+page.svelte` | Route target; renders `<Greeting />` |
| `Greeting.svelte` | Displays the greeting text; hosts one `$effect` rune |

## Technology Decisions (Rationale)

- **SvelteKit over raw Svelte:** Matches the Claude Cost stack pin in `CC_TRD`. Using the same framework for validation ensures the pipeline's framework handling is tested.
- **Svelte 5 runes over Svelte 4 stores:** Matches Claude Cost. Tests D2R's modern-Svelte handling.
- **TypeScript strict:** Catches type errors pre-build; matches Martinez Methods default for all code.
- **Static adapter:** No server runtime needed; simplifies the validation (no hosting questions).

## Deployment Architecture

Local preview only for validation purposes. The built `build/` directory is served via `npm run preview` on port 4173 for measurement-tool access (axe-core, Lighthouse). No cloud deployment needed.

## Observability

None. The app is too small to warrant logging, metrics, or tracing. Console output during `$effect` execution is the only observable runtime signal.

## Scalability

N/A. Static site.

## Disaster Recovery

N/A. No data to recover.

N/A. This is a validation-harness app; not intended to evolve into production.
